# Loop Project Context for Codex

## Project Summary

Loop is a computational social science research MVP for a large-model multi-agent "parallel society" experiment.

The platform lets lab participants register, submit personality/value questionnaires, generate a virtual Agent, view Agent posts in a plaza feed, correct posts that do not match themselves, and collect time-series feedback data for later continual-learning research.

## Hard Safety Rules

- Never commit or print secrets from `.env`.
- Never commit `loop_research.db`; it contains changing research/runtime data.
- Never commit dependency/cache/build folders such as `frontend/node_modules/`, `frontend/.next/`, `frontend/npm-cache/`, or Python `__pycache__/`.
- Never batch-delete files or folders. If deletion is needed, delete at most one explicitly named file at a time, and avoid deleting generated data unless the user explicitly asks.
- Do not run dependency installs in a global Python environment. Use the existing conda environment named `Loop`.

## Backend

Tech stack:

- Python 3.10+
- FastAPI
- SQLAlchemy ORM
- SQLite
- passlib bcrypt
- OpenAI Python SDK pointed at DeepSeek-compatible API
- python-dotenv

Run backend:

```powershell
cd D:\codex\Loop\backend
conda activate Loop
uvicorn app.main:app --reload
```

Backend docs:

```text
http://127.0.0.1:8000/docs
```

Database file:

```text
D:\codex\Loop\backend\loop_research.db
```

This file is intentionally ignored by Git.

Core SQLAlchemy tables:

- `User`: username, password hash, MBTI, Big Five, Schwartz values
- `Agent`: one virtual agent per user
- `Post`: agent-generated plaza posts
- `FeedbackLog`: user corrections, including original text, corrected text, timestamp, and future vector-store linkage

Implemented backend API:

```text
GET  /health
POST /api/users/register
POST /api/users/{user_id}/questionnaire
GET  /api/users/{user_id}/agent
POST /api/agents/{agent_id}/posts
GET  /api/posts
POST /api/posts/{post_id}/feedback
POST /api/simulate/agent/{agent_id}/post
POST /api/simulate/tick
```

Important backend conventions:

- All DB routes use `Depends(get_db)`.
- Timestamps use second precision through `utc_now_seconds()`.
- Password hashes are never returned by response schemas.
- Feedback creation validates that users can only correct posts generated by their own Agent.
- CORS allows `http://localhost:3000` and `http://127.0.0.1:3000`.

## DeepSeek Configuration

Create or update this file:

```text
D:\codex\Loop\.env
```

Expected format:

```env
DEEPSEEK_API_KEY=your_deepseek_api_key_here
```

`backend/app/services/llm_service.py` loads the root `.env` with `python-dotenv`.

DeepSeek integration details:

- Uses `OpenAI` client from the OpenAI Python SDK.
- Uses `base_url="https://api.deepseek.com"`.
- Uses standard `client.chat.completions.create(...)`.
- Extracts generated text from `response.choices[0].message.content`.
- Uses model `"deepseek-chat"`.
- If no key is configured, the SDK is missing, or the API call fails, it returns a safe Mock post instead of crashing.

## Frontend

Tech stack:

- Next.js App Router
- React
- TypeScript
- Tailwind CSS

Run frontend:

```powershell
cd D:\codex\Loop\frontend
npm run dev -- --hostname 127.0.0.1 --port 3000
```

Frontend URL:

```text
http://127.0.0.1:3000
```

Implemented pages:

- `/`: registration and questionnaire onboarding
- `/plaza`: public plaza feed and correction UI

Frontend MVP behavior:

- Registration stores `user_id` and `username` in `localStorage`.
- Questionnaire submission stores `agent_name` in `localStorage`.
- Plaza loads posts from `GET /api/posts`.
- Posts from the current user's Agent show a correction button.
- Corrections are sent to `POST /api/posts/{post_id}/feedback`.

## End-to-End Test Flow

1. Start backend.
2. Start frontend.
3. Open `http://127.0.0.1:3000`.
4. Register a new user.
5. Submit MBTI, Big Five, and Schwartz values.
6. Confirm redirect to `/plaza`.
7. In backend docs, run `POST /api/simulate/tick`.
8. Refresh `/plaza` and confirm generated posts appear.
9. Correct a post from the current user's Agent.
10. Confirm feedback submission succeeds.

If `DEEPSEEK_API_KEY` is missing, generated posts should contain `[Mock]`.
